## MODIFIED Requirements

### Requirement: 创建订单
系统 SHALL 允许已登录买家将购物车中选中的商品创建为一笔订单，并扣减库存；创建订单时如果买家拥有有效平台会员，系统 MUST 自动应用会员折扣并保存会员优惠快照；创建订单时如果买家提交优惠券，系统 MUST 按优惠券叠加配置校验并保存全部优惠券使用快照。

#### Scenario: 普通买家下单成功
- **WHEN** 非会员买家提交包含收货地址的下单请求
- **THEN** 系统创建订单（状态：PENDING_PAYMENT），扣减商品库存，清空已购商品的购物车条目，返回订单ID
- **AND** 订单会员优惠金额为 0

#### Scenario: 有效会员下单成功
- **WHEN** 有效会员买家提交包含收货地址的下单请求
- **THEN** 系统创建订单（状态：PENDING_PAYMENT），扣减商品库存，清空已购商品的购物车条目，返回订单ID
- **AND** 系统按会员套餐折扣率计算会员优惠金额和实际支付金额
- **AND** 系统保存会员套餐 ID、套餐名称、折扣率和会员优惠金额快照

#### Scenario: 单张优惠券下单
- **WHEN** 买家提交包含收货地址和一张买家优惠券 ID 的下单请求
- **AND** 该优惠券符合订单使用资格
- **THEN** 系统创建订单并记录该优惠券使用明细
- **AND** 系统将该买家优惠券标记为 `USED`

#### Scenario: 会员专属券与普通券叠加下单
- **WHEN** 买家提交包含收货地址、一张普通优惠券 ID 和一张会员专属优惠券 ID 的下单请求
- **AND** 两张优惠券符合叠加使用资格
- **THEN** 系统先计算优惠券总抵扣金额
- **AND** 系统保存两张优惠券的名称、适用人群、门槛、模板抵扣金额、实际抵扣金额和叠加配置快照
- **AND** 系统将两张买家优惠券均标记为 `USED`

#### Scenario: 会员折扣与叠加优惠券共同下单
- **WHEN** 有效会员买家提交包含收货地址、一张普通优惠券 ID 和一张会员专属优惠券 ID 的下单请求
- **AND** 两张优惠券符合叠加使用资格
- **THEN** 系统先计算优惠券总抵扣金额
- **AND** 系统再对优惠券抵扣后的剩余金额应用会员折扣
- **AND** 系统保存订单原始金额、优惠券优惠金额、会员优惠金额和实际支付金额

#### Scenario: 同时提交旧单券字段和新多券字段
- **WHEN** 买家在下单请求中同时提交旧字段 `buyerCouponId` 和新字段 `buyerCouponIds`
- **THEN** 系统返回 400 状态码
- **AND** 系统不创建订单、不扣减库存、不标记任何优惠券为已使用

#### Scenario: 会员套餐后续修改不影响订单
- **WHEN** 买家使用会员折扣创建订单后管理员修改、停用或删除该会员套餐
- **THEN** 已创建订单继续展示创建订单时保存的会员套餐名称、折扣率和会员优惠金额快照

#### Scenario: 优惠券模板后续修改不影响订单
- **WHEN** 买家使用优惠券创建订单后管理员修改优惠券名称、金额、状态或叠加配置
- **THEN** 已创建订单继续展示创建订单时保存的优惠券使用快照

#### Scenario: 过期会员下单不应用会员折扣
- **WHEN** 会员已过期的买家提交包含收货地址的下单请求
- **THEN** 系统按普通买家订单创建规则创建订单
- **AND** 订单会员优惠金额为 0

#### Scenario: 库存不足
- **WHEN** 下单时某商品库存不足
- **THEN** 系统返回 400 状态码，提示具体商品库存不足，不创建订单
- **AND** 系统不扣减库存、不清空购物车、不占用优惠券、不保存会员优惠快照

#### Scenario: 购物车为空下单
- **WHEN** 买家购物车为空时尝试下单
- **THEN** 系统返回 400 状态码并提示"请先添加商品到购物车"
- **AND** 系统不创建订单、不占用优惠券、不保存会员优惠快照

### Requirement: 订单优惠券占用释放
系统 SHALL 在使用优惠券的订单进入不再消耗优惠券的终态时释放该订单占用的全部买家优惠券，并保持订单金额与优惠券快照不变。

#### Scenario: 待付款订单取消退回全部优惠券
- **WHEN** 买家取消一笔状态为 `PENDING_PAYMENT` 且已使用一张或多张买家优惠券的订单
- **THEN** 系统将订单状态更新为 `CANCELLED`
- **AND** 系统恢复订单商品库存
- **AND** 系统将该订单占用的全部买家优惠券恢复为可再次使用的状态，并清除使用订单 ID 和使用时间
- **AND** 系统保留订单原始金额、优惠金额、实际支付金额和优惠券快照

#### Scenario: 支付超时自动取消退回全部优惠券
- **WHEN** 系统自动取消一笔超过支付截止时间、状态为 `PENDING_PAYMENT` 且已使用一张或多张买家优惠券的订单
- **THEN** 系统将订单状态更新为 `CANCELLED`
- **AND** 系统恢复订单商品库存
- **AND** 系统将该订单占用的全部买家优惠券恢复为可再次使用的状态，并清除使用订单 ID 和使用时间

#### Scenario: 未使用优惠券的订单取消不触发优惠券退回
- **WHEN** 买家或系统取消一笔未使用优惠券的 `PENDING_PAYMENT` 订单
- **THEN** 系统将订单状态更新为 `CANCELLED` 并恢复库存
- **AND** 系统不修改任何买家优惠券记录

#### Scenario: 退款申请不退回优惠券
- **WHEN** 买家对一笔已使用优惠券且可退款的订单提交退款申请
- **THEN** 系统将订单状态更新为 `REFUND_REQUESTED`
- **AND** 该订单占用的买家优惠券保持 `USED`

#### Scenario: 退款批准退回全部优惠券
- **WHEN** 管理员批准一笔状态为 `REFUND_REQUESTED` 且已使用一张或多张买家优惠券的订单退款
- **AND** 支付渠道退款调用成功
- **THEN** 系统将订单状态更新为 `REFUNDED`
- **AND** 系统将该订单占用的全部买家优惠券恢复为可再次使用的状态，并清除使用订单 ID 和使用时间
- **AND** 系统保留订单原始金额、优惠金额、实际支付金额和优惠券快照

#### Scenario: 退款拒绝不退回优惠券
- **WHEN** 管理员拒绝一笔状态为 `REFUND_REQUESTED` 且已使用优惠券的订单退款
- **THEN** 系统将订单状态更新为 `REFUND_REJECTED`
- **AND** 该订单占用的买家优惠券保持 `USED`

#### Scenario: 支付渠道退款失败不退回优惠券
- **WHEN** 管理员批准退款但支付渠道退款调用失败
- **THEN** 系统保持订单状态为 `REFUND_REQUESTED`
- **AND** 该订单占用的买家优惠券保持 `USED`

#### Scenario: 已过期优惠券退回后不可再次用于下单
- **WHEN** 使用优惠券的订单取消或退款成功时，该优惠券活动已经超过有效期
- **THEN** 系统清除该买家优惠券的使用订单 ID 和使用时间
- **AND** 买家优惠券列表将该券展示为 `EXPIRED`
- **AND** 该券不会出现在结算页可用优惠券列表中

#### Scenario: 优惠券退回幂等
- **WHEN** 同一笔已使用优惠券的订单被重复触发取消或退款成功后的退回逻辑
- **THEN** 系统最多退回一次该订单占用的每张买家优惠券
- **AND** 系统不修改未被该订单占用的其他买家优惠券记录
