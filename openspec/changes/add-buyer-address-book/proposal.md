## 背景与动机

买家的收货地址通常固定在家、公司等少数几个地点，但当前每次购买都需要重新填写完整地址，重复成本高，也容易造成输入错误。引入买家地址簿后，买家可以保存常用地址，在结算时直接选择并填入，同时继续沿用服务端地址校验以保证经纬度和地址有效性。

## 变更内容

- 新增买家地址簿能力：买家可以新增、编辑、删除、查看自己的常用收货地址。
- 支持默认地址：买家可以设置一个默认地址，结算页优先展示并填入默认地址。
- 新增结算页地址选择：买家下单时可以从地址簿选择地址并自动填入收货人、手机号、省市区、详细地址等字段。
- 地址保存时继续走服务端地图地址校验：保存成功后记录经纬度、校验状态和可选的标准化地址。
- 下单时仍以服务端校验/已校验地址为准，不信任前端直接传入的经纬度或 `VALID` 状态。
- 历史订单地址快照保持不变；地址簿后续修改不影响已创建订单。

## 能力范围

### 新增能力

- `buyer-address-book`: 买家常用收货地址的保存、编辑、删除、默认地址、地址校验和结算页选择填入。

### 修改能力

- `order-management`: 创建订单时支持使用已保存地址，结算页可以从地址簿选择并填入收货地址，同时继续保留手动输入地址能力。

## 影响范围

- 后端：新增买家地址实体、repository、service、controller、请求/响应 DTO；复用 `AddressValidationService`；更新下单请求以支持 `addressId`。
- 前端：新增地址管理入口/页面或组件；更新结算页地址选择、默认地址自动填入和保存新地址交互。
- 数据库：新增买家地址表，保存结构化地址、经纬度、校验状态、默认标记、创建/更新时间。
- 权限：地址簿接口仅允许买家访问，买家只能管理自己的地址。
- 测试：需要覆盖地址 CRUD、默认地址唯一性、权限隔离、地址校验失败、使用地址簿下单、地址快照不受后续修改影响，以及前端构建验证。
