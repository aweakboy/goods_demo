## MODIFIED Requirements

### Requirement: 卖家订单管理
系统 SHALL 允许卖家查看收到的全部订单、创建发货记录、查看物流状态、物流轨迹、可用的缓存路径规划路线，并处理物流异常。

#### Scenario: 查看收到的订单
- **WHEN** 卖家访问订单管理页面
- **THEN** 系统返回包含该卖家商品的所有订单，按创建时间倒序，并包含物流摘要和可用的物流地图数据

#### Scenario: 标记发货
- **WHEN** 卖家对 PAID 状态订单点击发货并填写物流公司和运单号
- **THEN** 系统创建 shipment，将订单状态更新为 SHIPPED，记录物流信息和初始轨迹，并在坐标完整且路径规划可用时保存路线快照

#### Scenario: 发货时路径规划失败
- **WHEN** 卖家标记发货时外部路径规划失败或未配置
- **THEN** 系统仍创建 shipment 并更新订单为 SHIPPED，页面展示物流摘要、时间线和路线降级说明

#### Scenario: 查看缓存路线
- **WHEN** 卖家查看存在 shipment 且路线快照可用的已发货订单
- **THEN** 页面展示发货地、收货地、真实道路路线、已完成路线段和当前位置 marker

#### Scenario: 地图不可用降级
- **WHEN** 卖家订单缺少坐标、路线快照不可用、地图 Key 未配置或地图 SDK 加载失败
- **THEN** 页面继续展示文本物流摘要和轨迹时间线，不阻断订单操作

#### Scenario: 查看物流异常
- **WHEN** 卖家查看 shipment 状态为 EXCEPTION 的订单
- **THEN** 页面突出显示异常状态、最近一条异常轨迹说明，并在地图上保留最近可计算位置

#### Scenario: 按状态筛选订单
- **WHEN** 卖家选择指定订单状态筛选
- **THEN** 系统只返回该状态的订单列表
