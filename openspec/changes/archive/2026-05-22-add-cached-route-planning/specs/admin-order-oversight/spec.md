## MODIFIED Requirements

### Requirement: 查看订单详情
系统 SHALL 允许管理员查看任意订单的完整详情，包括订单基础信息、收货地址、商品条目、支付/订单状态、物流摘要、物流轨迹、可用的缓存路径规划路线，并在必要时刷新路线规划。

#### Scenario: 查看订单详情
- **WHEN** 管理员点击某笔订单查看详情
- **THEN** 系统返回订单完整信息：订单ID、买家信息、收货地址、所有商品条目（名称/单价/数量）、总金额、状态、物流公司、物流单号、物流状态、物流轨迹、物流地图数据、创建时间

#### Scenario: 查看缓存路线
- **WHEN** 管理员查看存在 shipment 且路线快照可用的订单详情
- **THEN** 页面展示物流地图、发货地 marker、收货地 marker、真实道路路线、已完成路线段、当前位置 marker、距离和预计耗时

#### Scenario: 查看路线降级订单详情
- **WHEN** 管理员查看存在 shipment 但路线快照不可用的订单详情
- **THEN** 页面展示物流轨迹和模拟路线或降级原因，不触发外部路径规划调用

#### Scenario: 管理员刷新路线
- **WHEN** 管理员在订单详情中手动刷新 shipment 路线规划且刷新成功
- **THEN** 页面刷新路线状态、真实道路路线、距离、预计耗时和当前位置 marker

#### Scenario: 管理员刷新路线失败
- **WHEN** 管理员手动刷新 shipment 路线规划失败
- **THEN** 页面展示刷新失败原因，并保留旧的可用路线或降级路线

#### Scenario: 推进物流后刷新地图
- **WHEN** 管理员点击模拟推进物流
- **THEN** 页面刷新物流状态、轨迹时间线和基于当前路线来源计算的地图当前位置 marker

#### Scenario: 查看物流异常订单详情
- **WHEN** 管理员查看 shipment 状态为 EXCEPTION 的订单详情
- **THEN** 系统返回异常状态、异常原因轨迹、最近更新时间和异常位置地图数据

#### Scenario: 查看历史物流订单详情
- **WHEN** 管理员查看没有 shipment 但有旧物流单号的订单详情
- **THEN** 系统展示旧物流单号，并标记暂无结构化物流轨迹和地图路线
