## MODIFIED Requirements

### Requirement: 卖家订单管理
系统 SHALL 允许卖家查看收到的全部订单、创建发货记录、查看物流状态、物流轨迹和可用的地图路线模拟，并处理物流异常。

#### Scenario: 查看收到的订单
- **WHEN** 卖家访问订单管理页面
- **THEN** 系统返回包含该卖家商品的所有订单，按创建时间倒序，并包含物流摘要和可用的物流地图模拟数据

#### Scenario: 标记发货
- **WHEN** 卖家对 PAID 状态订单点击发货并填写物流公司和运单号
- **THEN** 系统创建 shipment，将订单状态更新为 SHIPPED，并记录物流信息、初始轨迹和可用的地图路线模拟数据

#### Scenario: 查看物流地图
- **WHEN** 卖家查看存在 shipment 且坐标完整的已发货订单
- **THEN** 页面展示发货地、收货地、模拟路线和当前位置 marker

#### Scenario: 地图不可用降级
- **WHEN** 卖家订单缺少坐标、地图 Key 未配置或地图 SDK 加载失败
- **THEN** 页面继续展示文本物流摘要和轨迹时间线，不阻断订单操作

#### Scenario: 查看物流异常
- **WHEN** 卖家查看 shipment 状态为 EXCEPTION 的订单
- **THEN** 页面突出显示异常状态、最近一条异常轨迹说明，并在地图上保留最近模拟位置

#### Scenario: 按状态筛选订单
- **WHEN** 卖家选择指定订单状态筛选
- **THEN** 系统只返回该状态的订单列表
