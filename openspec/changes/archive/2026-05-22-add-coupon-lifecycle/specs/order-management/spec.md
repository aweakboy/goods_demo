## MODIFIED Requirements

### Requirement: 创建订单
系统 SHALL 允许已登录买家将购物车中选中的商品创建为一笔订单，并扣减库存；创建订单时可以不使用优惠券，也可以使用当前买家自己的可用优惠券抵扣订单金额。

#### Scenario: 不使用优惠券下单成功
- **WHEN** 买家提交包含收货地址且不包含优惠券的下单请求
- **THEN** 系统创建订单（状态：PENDING_PAYMENT），扣减商品库存，清空已购商品的购物车条目，返回订单ID
- **AND** 订单原始金额等于实际支付金额，优惠金额为 0

#### Scenario: 使用优惠券下单成功
- **WHEN** 买家提交包含收货地址和 `buyerCouponId` 的下单请求
- **AND** 该优惠券属于当前买家、状态为 `UNUSED`、未过有效期、活动状态为 `ACTIVE` 且订单商品原始金额达到满减门槛
- **THEN** 系统创建订单（状态：PENDING_PAYMENT），扣减商品库存，清空已购商品的购物车条目，返回订单ID
- **AND** 系统保存订单原始金额、优惠金额、实际支付金额和优惠券快照
- **AND** 系统将该买家优惠券标记为 `USED` 并记录使用订单 ID

#### Scenario: 优惠券后续修改不影响订单
- **WHEN** 买家使用优惠券创建订单后管理员修改、停用或删除该优惠券活动
- **THEN** 已创建订单继续展示创建订单时保存的优惠券名称、门槛金额、抵扣金额和订单金额快照

#### Scenario: 使用无效优惠券下单
- **WHEN** 买家提交不存在、不属于自己、已使用、已过期、未启用或未达到满减门槛的 `buyerCouponId`
- **THEN** 系统返回对应的 400、403 或 404 状态码
- **AND** 系统不创建订单、不扣减库存、不清空购物车、不标记优惠券为已使用

#### Scenario: 库存不足
- **WHEN** 下单时某商品库存不足
- **THEN** 系统返回 400 状态码，提示具体商品库存不足，不创建订单
- **AND** 如请求包含优惠券，系统不标记优惠券为已使用

#### Scenario: 购物车为空下单
- **WHEN** 买家购物车为空时尝试下单
- **THEN** 系统返回 400 状态码并提示"请先添加商品到购物车"
- **AND** 如请求包含优惠券，系统不标记优惠券为已使用
