## Context

当前平台已经有买家、订单、支付宝沙箱支付、优惠券领取和使用能力，但缺少“买家付费成为平台会员后获得持续权益”的模型。会员功能会跨越支付、订单金额计算、优惠券资产、管理后台和买家前端，因此需要明确会员身份、续期、权益发放和下单优惠的边界。

本变更默认依赖 `add-coupon-lifecycle` 的优惠券模型：平台优惠券模板和买家优惠券实例已经存在。会员每月专属券复用该模型，但需要增加会员专属领取入口和限制。若实现时 `add-coupon-lifecycle` 尚未归档，应先保证优惠券基础表、服务和接口已经可用，再实现会员权益。

## Goals / Non-Goals

**Goals:**

- 管理员可以配置平台会员套餐，包括价格、有效期、会员折扣率、每月专属优惠券模板和启停状态。
- 买家可以购买或续费平台会员，支付成功后激活或延长会员有效期。
- 买家可以查看自己的会员状态、到期时间、当前套餐和当月权益领取状态。
- 有效会员每个自然月可以领取一次套餐配置的会员专属优惠券，非会员和过期会员不可领取。
- 有效会员下单时自动应用会员折扣，并在订单上保存会员套餐、折扣率、会员优惠金额等快照。
- 会员折扣可以和优惠券叠加使用，最终金额必须以后端计算为准，并且不能出现负数金额。
- 用自动化测试覆盖会员购买、续期、到期、月度权益、订单金额、权限和异常回滚。

**Non-Goals:**

- 不支持多等级成长值、积分、储值余额、自动扣费续费或会员等级升级差价补缴。
- 不支持卖家自定义会员权益；本变更只做平台级会员。
- 不支持部分月份按天折算权益；会员专属券按自然月领取。
- 不实现复杂营销叠加规则引擎；只定义一个固定的会员折扣和优惠券叠加顺序。
- 不修复历史订单或历史优惠券数据。

## Decisions

### D1：使用“会员套餐 + 买家会员 + 会员购买记录”三类模型

新增 `membership_plans` 保存后台配置，关键字段包括：

- `name`、`description`
- `price`
- `duration_months`
- `discount_rate`：如 `0.95` 表示 95 折
- `monthly_coupon_id`：每月会员专属券模板，可为空
- `status`：`ACTIVE`、`INACTIVE`
- `created_at`、`updated_at`

新增 `buyer_memberships` 保存买家当前会员身份，关键字段包括：

- `buyer_id`
- `plan_id`
- `status`：`ACTIVE`、`EXPIRED`
- `started_at`、`expires_at`
- `last_paid_at`

新增 `membership_orders` 或 `membership_purchases` 保存购买记录，关键字段包括：

- `buyer_id`、`plan_id`
- `amount`
- `status`：`PENDING_PAYMENT`、`PAID`、`CANCELLED`、`EXPIRED`
- `out_trade_no`、`alipay_trade_no`
- `paid_at`、`created_at`

原因：套餐是后台运营配置，买家会员是当前权益状态，购买记录用于支付、对账和续期审计。把这些概念拆开可以避免订单支付回调和会员状态互相污染。

备选方案是在 `User` 表上直接添加会员字段。该方案难以记录购买历史、续费和套餐快照，因此不采用。

### D2：会员支付使用独立购买记录，支付回调按业务类型路由

买家购买会员时创建会员购买记录并发起支付。支付成功后，会员服务根据购买记录激活或延长会员，不直接复用商品订单表。

支付流水号应带业务前缀或业务类型，例如 `MEM-<purchaseId>`，支付回调通过前缀或持久化映射识别是商品订单还是会员购买。

原因：会员购买不是商品订单，不涉及库存、收货地址、发货和退款状态机。独立购买记录可以降低对 `orders` 表的语义污染。

### D3：续费从较晚时间开始延长

买家已有有效会员时再次购买同一平台会员，新的有效期从当前 `expiresAt` 起延长。买家会员已过期时，新的有效期从支付成功时间开始计算。

原因：该规则符合用户对“未到期续费不损失剩余天数”的预期，并且实现简单、可测试。

### D4：月度权益按自然月去重

新增 `membership_monthly_benefits` 记录买家每月权益领取状态，关键字段包括：

- `buyer_id`
- `plan_id`
- `benefit_month`：格式如 `2026-05`
- `coupon_id`
- `buyer_coupon_id`
- `claimed_at`

同一买家同一 `benefit_month` 对同一会员套餐最多成功领取一次。领取时校验买家会员有效、当前时间在会员有效期内、套餐仍启用，且套餐配置了月度优惠券模板。

原因：用唯一约束保护重复点击和并发请求，避免同月重复发券。

### D5：会员专属券复用优惠券模板和买家优惠券实例

会员每月领取权益时，由会员服务调用优惠券服务创建买家优惠券实例。会员专属券不应出现在普通可领取列表中，非会员不可通过普通领取接口绕过限制。

实现上可在优惠券模板增加 `audience` 字段（如 `PUBLIC`、`MEMBER`）和可选 `membership_plan_id`，或者在会员套餐中保存专属优惠券模板 ID 并由会员服务绕过普通公开领取入口。优先选择显式 `audience` 字段，因为它可以让列表和权限判断更清晰。

### D6：订单金额计算保存会员快照

订单创建时，后端读取买家当前有效会员。若存在有效会员：

1. 计算商品原始金额 `originalAmount`。
2. 按现有优惠券规则校验并计算优惠券抵扣。
3. 对优惠券抵扣后的剩余金额应用会员折扣率，计算 `membershipDiscountAmount`。
4. `totalAmount = max(0, originalAmount - couponDiscountAmount - membershipDiscountAmount)`。

订单新增会员快照字段：

- `membership_plan_id`
- `membership_plan_name`
- `membership_discount_rate`
- `membership_discount_amount`

未使用会员折扣时，`membership_discount_amount = 0`。订单详情和后台订单列表读取订单快照，不受套餐后续改价、停用或折扣调整影响。

原因：先使用优惠券再使用会员折扣，可以复用现有优惠券“按订单商品原始金额校验门槛”的规则，并避免优惠券状态机变化。会员折扣只影响最终应付金额，不改变优惠券领取和使用资格。

## Risks / Trade-offs

- **[Risk] 支付回调重复通知导致重复续期** → 会员购买记录以 `PAID` 状态做幂等保护；已支付记录再次收到回调不重复延长会员。
- **[Risk] 月度权益重复领取** → `membership_monthly_benefits` 添加买家、套餐、月份唯一约束，并在服务层事务内创建记录和买家优惠券。
- **[Risk] 优惠券普通入口绕过会员限制** → 会员专属券必须在优惠券列表和领取接口中排除或校验 `audience`。
- **[Risk] 订单金额计算和前端预览不一致** → 前端只展示预览，最终金额以后端创建订单返回和订单详情为准；服务测试覆盖计算公式。
- **[Risk] 套餐变更影响历史订单** → 订单保存会员折扣快照，购买记录保存支付金额，历史展示不读取当前套餐配置。
- **[Risk] 多个未归档 change 交叉影响** → 实现前确认 `add-coupon-lifecycle` 和支付相关基础代码已经存在；若字段冲突，应优先复用已有模型而不是重复建表。

## Migration Plan

- 新增会员相关表和订单会员快照字段，默认历史订单 `membership_discount_amount = 0`。
- 优惠券模板若新增 `audience` 字段，历史优惠券默认设为 `PUBLIC`。
- 部署后管理员先创建并启用会员套餐，再开放买家购买入口。
- 回滚时可以隐藏前端会员入口并停用会员套餐；已生成的会员购买记录和订单快照保留。

## Testing Strategy

- 后端 service 测试覆盖套餐创建/启停、无效价格/折扣校验、非管理员权限。
- 后端 service 测试覆盖会员购买支付成功激活、有效期内续费延长、过期后重新购买、重复支付回调幂等。
- 后端 service 测试覆盖有效会员领取当月专属券、非会员不可领取、过期会员不可领取、同月重复领取失败、并发重复领取不重复发券。
- 后端订单测试覆盖非会员无会员折扣、有效会员自动应用折扣、会员折扣与优惠券叠加、套餐后续修改不影响订单快照。
- 后端 API/controller 测试覆盖管理员套餐接口、买家会员接口和跨角色访问返回 403。
- 前端当前没有测试 runner，前端变更至少运行 `npm.cmd run build`，并保留会员购买、权益领取、结算金额展示的手动验证任务。
- 验证命令包括 `mvn.cmd test`、`npm.cmd run build` 和 `openspec validate add-membership-benefits`。

## Open Questions
