## Why

当前系统已经有结构化店铺地址和订单收货地址字段，但这些地址仍只依赖用户输入，无法确认是否真实可定位，也无法稳定用于地图展示、距离估算和后续物流模拟。
接入地图服务并完成地址校验，可以把“文本地址”升级为“可定位地址”，为真实发货场景建立基础。

## What Changes

- 接入高德地图作为默认地图服务：后端使用 Web 服务 API 做地址解析/地理编码，前端使用 JS API 展示地图。
- 新增后端地址校验服务，根据省、市、区、详细地址解析经纬度，并校验返回结果与用户提交的行政区一致。
- 店铺注册/编辑时，服务端校验店铺地址，成功后保存经纬度和 `addressValidationStatus=VALID`；无匹配或不一致时拒绝保存。
- 买家下单时，服务端校验收货地址，成功后将经纬度和校验状态保存到订单地址快照；无匹配或不一致时拒绝下单。
- 前端店铺地址表单和结算页提供地址校验/地图预览体验，避免用户提交无法定位的地址。
- 店铺主页、订单详情、卖家订单/管理员订单详情在有经纬度时展示地图定位信息。
- 保留历史数据兼容：已有 `UNVERIFIED` 地址仍可展示文本，但不会被当作已校验地址。

## Capabilities

### New Capabilities

- `map-address-validation`: 地图服务配置、地址解析、地址校验、经纬度返回和地图展示基础能力。

### Modified Capabilities

- `seller-shop-management`: 店铺注册和编辑必须通过服务端地址校验并保存经纬度。
- `order-management`: 创建订单必须通过服务端收货地址校验，并保存收货地址快照经纬度。
- `shop-storefront`: 店铺主页在店铺地址已校验且有经纬度时展示地图定位。
- `seller-dashboard`: 卖家订单管理在收货地址已校验且有经纬度时展示收货定位。
- `admin-order-oversight`: 管理员订单详情在收货地址已校验且有经纬度时展示收货定位。

## Impact

- 后端：新增地图服务配置、地图服务客户端、地址校验 service/DTO；更新 `ShopService`、`OrderService`、相关响应 DTO 和测试。
- 前端：新增地图加载封装和地图组件；更新 `SellerShopView.vue`、`CheckoutView.vue`、`ShopStorefrontView.vue`、订单详情/订单管理页面。
- 配置：新增高德 Web 服务 Key、JS API Key、安全密钥等环境配置；本地开发需要在环境变量或配置文件中提供。
- 数据库：复用已有经纬度和地址校验状态字段，必要时可补充 provider/adcode/校验消息字段；历史数据允许保持 `UNVERIFIED`。
- 外部系统：高德地图 Web 服务 API 和 JS API；需要考虑网络失败、配额限制和 Key 类型匹配。
