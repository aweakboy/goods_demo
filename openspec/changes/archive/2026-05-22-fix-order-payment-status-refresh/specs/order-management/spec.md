## ADDED Requirements

### Requirement: 支付状态一致性
系统 SHALL 保证支付平台已成功扣款的订单能够在本地订单系统中被确认并更新为已付款；当前端或回调流程发现订单仍为待付款时，系统 MUST 通过服务端向支付平台核验真实交易状态，不得仅依赖前端展示刷新。

#### Scenario: 支付通知成功更新订单
- **WHEN** 系统收到验签通过且交易状态为 `TRADE_SUCCESS` 或 `TRADE_FINISHED` 的支付通知
- **THEN** 系统将对应 `PENDING_PAYMENT` 订单更新为 `PAID`
- **AND** 系统记录支付平台交易流水号

#### Scenario: 支付通知未落库时主动核验成功
- **WHEN** 买家已在支付平台完成支付但本地订单仍为 `PENDING_PAYMENT`
- **AND** 买家触发支付状态核验
- **THEN** 系统通过服务端向支付平台查询该订单交易状态
- **AND** 当交易状态成功且订单号、金额校验通过时，系统将订单更新为 `PAID`

#### Scenario: 支付金额不匹配时拒绝入账
- **WHEN** 系统主动查询到支付平台交易成功
- **AND** 支付平台返回金额与本地订单应付金额不一致
- **THEN** 系统不得将订单更新为 `PAID`
- **AND** 系统返回支付状态无法确认的结果

#### Scenario: 重复成功处理保持幂等
- **WHEN** 系统重复收到同一已付款订单的成功通知或主动核验请求
- **THEN** 系统返回成功处理结果
- **AND** 订单状态保持 `PAID`

#### Scenario: 非成功交易不更新订单
- **WHEN** 支付通知或主动查询得到的交易状态不是成功或已完成
- **THEN** 系统不更新订单状态

#### Scenario: 验签失败不更新订单
- **WHEN** 系统收到验签失败的支付通知
- **THEN** 系统不更新订单状态
- **AND** 系统向支付平台返回失败处理结果

### Requirement: 支付后状态展示
系统 SHALL 在支付完成返回平台后展示本地后端确认后的订单状态；前端 MUST 通过后端接口获取或触发支付状态核验，不得自行把订单显示为已付款。

#### Scenario: 支付结果页确认已付款
- **WHEN** 买家从支付平台返回支付结果页
- **AND** 后端确认订单已经支付成功
- **THEN** 前端展示支付成功和已付款状态
- **AND** 前端提供进入该订单详情或订单列表的操作

#### Scenario: 支付结果页仍在确认中
- **WHEN** 买家返回支付结果页但后端暂未确认订单为 `PAID`
- **THEN** 前端展示支付确认中
- **AND** 前端在有界时间内继续触发后端核验或刷新订单状态

#### Scenario: 订单详情页支付后自动核验
- **WHEN** 买家从订单详情页发起支付并回到该页面
- **THEN** 前端触发后端支付状态核验或刷新订单详情
- **AND** 当后端返回订单状态为 `PAID` 时，前端隐藏支付入口并展示已付款状态
